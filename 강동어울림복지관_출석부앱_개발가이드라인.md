## 1. 전체 아키텍처 이해

```
┌─────────────────────────────────────────────────────────────┐
│                        사용자 디바이스                        │
│          스마트폰 / 태블릿 / PC (반응형 웹앱)                │
└──────────────────────┬──────────────────────────────────────┘
                       │  HTTPS 요청
┌──────────────────────▼──────────────────────────────────────┐
│               GitHub Pages (프론트엔드)                      │
│   index.html / dashboard.html / attendance.html 등          │
│   css/style.css · js/api.js · js/auth.js · js/attendance.js│
└──────────────────────┬──────────────────────────────────────┘
                       │  fetch() → GAS 웹앱 URL
┌──────────────────────▼──────────────────────────────────────┐
│          Google Apps Script (GAS) 백엔드                     │
│   Code.gs — doPost/doGet → 비즈니스 로직 처리               │
│   JWT 인증 · 출석 저장 · 실적 집계 · QR 토큰 발급           │
└──────────────────────┬──────────────────────────────────────┘
                       │  SpreadsheetApp API
┌──────────────────────▼──────────────────────────────────────┐
│             Google Sheets (데이터베이스)                     │
│   사업_마스터 / 회원_마스터 / 출석_원장                     │
│   실적_집계 / 직원_마스터                                    │
└─────────────────────────────────────────────────────────────┘
```

**핵심 원칙**

- 별도 서버 없음 → 모든 백엔드 로직은 GAS에서 처리
- 데이터는 Google Sheets에 저장 → 직원이 시트에서 직접 확인·수정 가능
- 프론트엔드는 정적 파일만으로 구성 → GitHub Pages 무료 호스팅

## STEP 1 — Google Sheets 세팅

### 목표
출석부 시스템의 데이터베이스 역할을 할 Google Sheets를 생성하고, 5개 시트를 구성합니다.

### 진행 순서

**1-1. 새 스프레드시트 생성**

```
1. Google Drive 접속 (drive.google.com)
2. [+ 새로 만들기] → [Google 스프레드시트]
3. 제목 변경: "강동어울림복지관_출석부DB"
4. URL에서 스프레드시트 ID 복사 (나중에 GAS에 입력)
   예: https://docs.google.com/spreadsheets/d/[이 부분이 ID]/edit
```

**1-2. 시트 5개 생성 (이름 정확히 입력)**

```
시트1: 사업_마스터
시트2: 회원_마스터
시트3: 출석_원장
시트4: 실적_집계
시트5: 직원_마스터
```

> 시트 탭 우클릭 → 이름 바꾸기로 변경

**1-3. AI에게 헤더 행 자동 입력 요청**

아래 AI 요청 템플릿 사용:

```
[AI 요청]
Google Sheets에 다음 5개 시트의 헤더(1행)를 입력하는 GAS 코드를 작성해줘.
스프레드시트 ID는 [YOUR_SPREADSHEET_ID]야.

시트별 헤더:
- 사업_마스터: 팀명, 사업분류, 세부사업분류, 사업명, 실적유형, 상태,
  목표_건수, 목표_실인원, 목표_연인원, 담당자, 사업ID
- 회원_마스터: 이름, 시작일, 상태, 장애비장애구분, 메모
- 출석_원장: 출석ID, 날짜, 사업ID, 사업명, 팀명, 이름,
  출석여부, 건수, 입력방식, 입력자, 입력시각
- 실적_집계: 팀명, 사업명, 년도, 월, 실인원, 건수, 연인원,
  목표대비_실인원, 목표대비_건수, 목표대비_연인원
- 직원_마스터: 직원ID, 이름, 팀명, 직위, 이메일, 비밀번호해시,
  담당사업IDs, 상태

실행 후 각 시트 헤더 행을 파란색 배경 + 흰색 글씨로 스타일링하는 코드도 포함해줘.
```

**1-4. 사업 마스터 데이터 입력**

구축 프롬프트의 섹션 2에 있는 6개 팀 × 전체 사업 데이터를 입력합니다.

```
[AI 요청]
"강동어울림복지관 출석부 구축 프롬프트"의 섹션 2에 있는 
6개 팀 전체 사업 데이터를 사업_마스터 시트에 입력하는 GAS 코드를 작성해줘.
스프레드시트 ID: [YOUR_SPREADSHEET_ID]

사업ID는 PROG_001 형식으로 자동 생성하고,
상태는 기본값 "활성"으로 설정해줘.
```

**1-5. 완료 확인**

```
□ 5개 시트가 정상 생성됨
□ 사업_마스터 시트에 전체 사업 데이터 입력됨 (약 114개 행)
□ 헤더 스타일링 적용됨
□ 스프레드시트 ID 메모해둠
```

## STEP 2 — GAS 백엔드 구현

### 목표
Google Apps Script에서 출석부 앱의 모든 API를 구현합니다.

### 진행 순서

**2-1. Apps Script 에디터 열기**

```
1. 스프레드시트 상단 메뉴: [확장 프로그램] → [Apps Script]
2. 프로젝트 이름 변경: "강동어울림복지관_출석부API"
3. 기존 코드(function myFunction) 전체 삭제
```

**2-2. 환경 변수 설정 (중요)**

Apps Script 에디터에서:
```
왼쪽 사이드바 → [프로젝트 설정(톱니바퀴)] → [스크립트 속성]
다음 속성 추가:
  - SPREADSHEET_ID : [스프레드시트 ID]
  - JWT_SECRET     : [임의의 복잡한 문자열, 예: GDW2026@SecretKey!]
  - ALLOWED_ORIGIN : https://[GitHub계정].github.io
```

**2-3. AI에게 전체 Code.gs 요청**

```
[AI 요청]
강동어울림복지관 출석부 시스템의 Google Apps Script Code.gs 전체를 작성해줘.

[환경 정보]
- 스프레드시트 ID는 PropertiesService로 읽음 (키: SPREADSHEET_ID)
- JWT_SECRET도 PropertiesService로 읽음
- CORS 허용 오리진: PropertiesService의 ALLOWED_ORIGIN

[구현할 함수 목록]
1. doGet(e), doPost(e) — action 파라미터로 분기
2. 인증: login, register, verifyToken
3. 회원: getMembers, addMember, updateMember, deleteMember, importMembersCSV
4. 사업: getPrograms, addProgram, updateProgram, deleteProgram, importProgramsCSV
5. 출석: checkAttendance, getAttendanceSheet, updateAttendance,
         getQRToken, selfCheckIn, submitCountOnly
6. 실적: getStats, getAllStats, getPersonalStats, recalcStats

[비즈니스 로직]
- 실인원: 동일 사업+기간 내 중복 없는 회원 수
- 연인원: 출석O 레코드 총 합계
- 건수: 건수유형은 입력값 합계, 실인원유형은 별도 건수 필드 합계
- QR 토큰: 5분 유효, 스크립트 캐시에 저장

모든 응답은 JSON 형식: { success: true/false, data: ..., message: ... }
```

**2-4. 파일 구조 (다중 파일 필요 시)**

```
Code.gs          — 메인 라우터 (doGet, doPost)
Auth.gs          — 인증 관련 함수
Members.gs       — 회원 관리 함수
Programs.gs      — 사업 관리 함수
Attendance.gs    — 출석 관리 함수
Stats.gs         — 실적 집계 함수
Utils.gs         — 공통 유틸리티 (날짜 변환, ID 생성 등)
```

> GAS 에디터 좌측 [파일] → [+] → [스크립트]로 파일 추가

**2-5. 완료 확인**

```
□ Code.gs 이하 모든 .gs 파일 저장됨 (Ctrl+S)
□ 에디터에서 오류 없음 (빨간 밑줄 없음)
□ 스크립트 속성(환경변수) 3개 모두 입력됨
□ 테스트 실행: login 함수를 직접 실행하여 오류 없음 확인
```

## STEP 4 — 프론트엔드 구현

### 목표
8개 HTML 파일과 JS 모듈을 순서대로 구현합니다.

### 4-1. 공통 CSS 먼저 구현

```
[AI 요청]
강동어울림복지관 출석부 앱의 공통 CSS (css/style.css)를 작성해줘.

[디자인 요구사항]
- 글래스모피즘(Glassmorphism) 스타일
- 주요 색상: 흰색 + 파란색 계열 (채도가 너무 강하거나 심한 그라데이션 지양)
- 폰트: Paperlogy (CDN 없을 경우 Noto Sans KR 폴백)
- 모던하고 세련된 디자인
- 반응형 (모바일 360px ~ 데스크탑 1440px)

[포함할 컴포넌트 스타일]
- .glass-card (글래스 카드)
- .btn-primary, .btn-secondary, .btn-danger, .btn-ghost
- .form-input, .form-select, .form-label
- .badge (활성/보류/종결/건수유형/실인원유형)
- .table-glass (반응형 테이블)
- .nav-sidebar (사이드바 네비게이션)
- .topbar (상단 바)
- .modal-overlay, .modal-glass
- .toast (알림 메시지)
- .progress-bar (달성률 표시)
- .stat-card (실적 요약 카드)
- .attendance-grid (출석체크 그리드)
- 로딩 스피너

CSS 변수(--color-primary 등)를 :root에 정의하고
다크모드는 지원하지 않아도 됨
```

### 4-2. 공통 JS 모듈 구현

```
[AI 요청]
다음 4개 공통 JS 모듈을 순서대로 작성해줘.

1. js/api.js
   - fetchGAS(action, params, method='POST') 함수
   - Authorization 헤더에 JWT 토큰 자동 포함
   - 오류 처리 및 토스트 알림 연동

2. js/auth.js
   - login(team, name, password): GAS login 호출 → JWT 저장
   - logout(): 토큰 삭제 → index.html 이동
   - getUser(): 현재 로그인 사용자 정보 반환
   - requireAuth(): 비로그인 시 index.html 리다이렉트
   - hasRole(role): 역할 권한 확인 (관리자/팀장/팀원)

3. js/utils.js
   - formatDate(date): YYYY-MM-DD 형식
   - formatNumber(num): 천 단위 콤마
   - showToast(message, type): 성공/오류/경고 알림
   - showLoading() / hideLoading()
   - parseCSV(text): CSV 파싱
```

## 트러블슈팅

### 자주 발생하는 문제

**문제 1: GAS API 호출 오류 (CORS 에러)**
```
증상: 브라우저 콘솔에 "CORS policy" 오류
해결:
1. GAS 스크립트 속성의 ALLOWED_ORIGIN이 GitHub Pages URL과 일치하는지 확인
2. GAS doOptions 함수가 올바르게 구현되었는지 확인
3. GAS를 새 버전으로 재배포
```

**문제 2: 로그인 후 바로 로그아웃 됨**
```
증상: 로그인 성공했으나 대시보드에서 다시 로그인 화면으로 이동
해결:
1. 브라우저의 localStorage 허용 여부 확인
2. Safari 개인정보 보호 모드에서는 localStorage가 제한됨
3. Chrome 또는 일반 모드 사용 권장
```

**문제 3: QR코드가 스캔되지 않음**
```
증상: QR코드 이미지가 표시되나 스캔이 안 됨
해결:
1. qrcode.js CDN 로드 여부 확인 (콘솔에서 QRCode 객체 확인)
2. QR 이미지 크기가 너무 작은 경우 → CSS에서 크기 확대
3. 직접 URL로 접근 테스트: qr-checkin.html?token=xxx&program=xxx&date=xxx
```

**문제 4: 출석 저장 후 시트에 반영 안 됨**
```
증상: 저장 성공 토스트가 표시되나 시트에 데이터 없음
해결:
1. GAS 실행 로그 확인 ([실행] → [실행 로그])
2. 스프레드시트 ID가 스크립트 속성에 정확히 입력되었는지 확인
3. 시트 이름이 정확한지 확인 (한글 시트명 오타 주의)
```

**문제 5: CSV 업로드 후 인코딩 깨짐**
```
증상: 이름 등 한글이 깨져서 등록됨
해결:
1. CSV 파일을 UTF-8 (BOM 포함)으로 저장
   Excel: [다른 이름으로 저장] → [CSV UTF-8 (BOM)]
   메모장: [다른 이름으로 저장] → 인코딩: UTF-8 BOM
2. 헤더가 정확한지 확인: 이름, 시작일, 상태, 장애비장애구분, 메모
```
